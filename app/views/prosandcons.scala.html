@() @main("Pros & Cons") {

Pros

  • Pure type-safe Scala code checked at compile time
  • No magic, i.e. simple to understand
  • In bigger apps one can use Scala constructs to structure the DI code
  • Out of the box IDE support because it is pure Scala code
  • Dependency graph is constructed in one place

Cons

  • Maybe more boiler-plate than annotations
  • Just singleton scope (you could do prototype scope with by-name parameters)
